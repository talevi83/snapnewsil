# SnapNews IL - 🇮🇱 Israel News App

מערכת לצבירת חדשות, סיכום חכם באמצעות בינה מלאכותית, והצגת מידע רלוונטי על ישראל מהארץ ומהעולם.

## 📌 על הפרויקט

הפרויקט **SnapNewsIL** משמש כאגרגטור (אוסף) אתרי חדשות ישראליים ובינלאומיים עבור מגוון קטגוריות: חדשות, אקטואליה, צבא וביטחון, כלכלה ועוד. המערכת סורקת מקורות RSS (למשל: Ynet, N12, וואלה, מעריב, ועוד), מורידה את המידע, מאגדת סיפורים דומים, ובאמצעות ממשק מבוסס AI (OpenAI) היא מייצרת סיכומים חכמים לקורא.
בנוסף, יש במערכת סינון טכנולוגי מתקדם לצפייה במה שכותבים על ישראל בעולם ("World View Israel").

### תכונות עיקריות
- **אגרגציית RSS**: מושך כתבות באופן אוטומטי ממגוון הערוצים המקומיים והבינלאומיים.
- **סיכום חדשות (AI)**: שימוש במודל `gpt-4o` של OpenAI ליצירת תמצית איכותית ועניינית מתוך הכתבות.
- **זיהוי סיפורים דומים**: מקבץ כתבות שונות מאתרי חדשות שונים העוסקות באותו נושא (Jaccard similarity).
- **תרגום אוטומטי**: אפשרות תרגום כתבות בינלאומיות באמצעות AI.
- **פרוקסי ועיבוד תמונות**: מעקף חסימות (Hotlinking) לאתרי חדשות ועיבוד תמונות (כיווץ ואופטימיזציה ל-WebP) בעזרת הספרייה `sharp`.

---

## 🏗 ארכיטקטורה

- **Backend**: שרת Node.js מבוסס Express.js הפועל כ-API לצד הלקוח ולאיסוף החדשות.
- **Frontend**: עמוד HTML/CSS/JS (נמצא בתיקייה `public/`) המציג את הכתבות, מסנן לפי קטגוריות ותומך בממשק גרפי מודרני (כולל תמיכה במובייל/דסקטופ).
- **ספריות עיקריות בשרת**:
  - `rss-parser`: לאיבוד (Parsing) קובצי ה-RSS של ספקי החדשות.
  - `openai`: ממשק רשמי לפניות אל מודלי השפה של OpenAI.
  - `cheerio`: לצורך סריקת עמודי חדשות (Scraping) ושליפת התוכן המלא בצורה נקייה ומבונחת מהרעש (מודעות/תפריטים).
  - `sharp`: לעיבוד, כיווץ, ואופטימיזציה של תמונות על השרת תוך הקטנת נפח התעבורה.
  - `axios`: לביצוע קריאות שרת (HTTP Requests) כגון הבאת RSS או ביצוע Proxy של תמונות.
- **דוקר (Docker)**: קבצי `Dockerfile` ו-`docker-compose.yml` המאפשרים פריסה קלה בתוך קונטיינר, כולל ניהול גרסאות, ללא צורך בהגדרת סביבה פנימית על השרת המארח.

## 🔑 מפתחות נדרשים (API Keys)

בשביל שהפרויקט יעבוד בצורה מלאה, ובעיקר תכונות ה-AI ושירות "מהעולם על ישראל", יש לספק שני מפתחות API. צור קובץ בשם `.env` בתיקיית הבסיס (Root) של הפרויקט לפי המבנה הבא:

```env
PORT=3000
OPENAI_API_KEY=sk-...
NEWS_API_KEY=...
```

### 1. השגת מפתח OpenAI (עבור סיכום תכנים)
מפתח ה-API של OpenAI אחראי לסיכום הכתבות בעברית ולתרגום (גישה למודל GPT-4o).
1. היכנסו ל- [OpenAI Developer Platform](https://platform.openai.com/).
2. יש להירשם או להתחבר לחשבון.
3. נווטו בתפריט הנווט ל-**API Keys**.
4. לחצו על "Create new secret key".
5. העתיקו את המפתח והכניסו אותו בקובץ ה-`.env` לשדה `OPENAI_API_KEY`. (חובה לחבר אמצעי תשלום, שכן השירות עולה כסף לפי שימוש).

### 2. השגת מפתח NewsAPI (עבור לשונית "World View Israel")
NewsAPI משמש כדי להביא צבר כתבות מאתרי חדשות באנגלית דרך מסוף שמאפשר סינון מדויק לכל מה שנכתב על "Israel".
1. היכנסו לאתר [NewsAPI](https://newsapi.org/).
2. לחצו על "Get API Key" בלמעלה הימני שיהיה עליכם להזין פרטים בסיסיים על מנת להירשם.
3. לאחר ההרשמה תקבלו מפתח אלפאנומרי ארוך, אותו יש להדביק בתוך ה-`.env` תחת `NEWS_API_KEY`. (ישנו מסלול חינמי המגביל ל-100 קריאות ביום ולכתבות מחודש האחרון בלבד, שמספיק בשביל לפתח).

## 🚀 איך מפעילים?

ישנן שתי דרכים עקריות להפעלת הפרויקט: בעזרת Docker (קונטיינר) או הרצה מקומית באמצעות Node.js.

### אופציה א: הפעלה באמצעות Docker (הדרך המומלצת)

אם מותקן לכם [Docker](https://www.docker.com/) ו-Docker Compose במחשב או בשרת:
1. וודאו שהגדרתם את כל מפתחות ה-API בעורך הטקסט שבתוך קובץ ה-`docker-compose.yml` (תחת `environment`) או שימוש בקובץ ה-`.env`.
2. פתחו את שורת הפקודה (Terminal או CMD) בתיקייה הנוכחית.
3. הריצו:
   ```bash
   docker-compose up -d
   ```
4. הפרויקט יעבוד ברקע. כעת ניתן להיכנס בדפדפן ע"י הגלישה לכתובת: `http://localhost:3000`.
   כדי להפסיק את הריצה: `docker-compose down`.

### אופציה ב: הפעלה באמצעות Node.js באופן מקומי

אם אין לכם דוקר ואתם רוצים להריץ ישר על המחשב/שרת:
1. וודאו שברשותכם [Node.js](https://nodejs.org/) בגרסה 18 ומעלה.
2. פתחו שורת פקודה (Terminal) בתיקיית הפרויקט.
3. התקינו את ספריות הפיתוח שצוינו ב-`package.json` ע"י הפקודה:
   ```bash
   npm install
   ```
4. וודאו שקובץ `env.` עם מפתחות ה-API קיים בתיקייה.
5. הריצו את השרת:
   ```bash
   npm start
   ```
   *(במידה ואתם מפתחים של הפרויקט עדיף להריץ במצב צופה `npm run dev` שיאזר ב-nodemon לרענונים חיים)*.
6. גילשו בעזרת הדפדפן לכתובת: `http://localhost:3000`.
